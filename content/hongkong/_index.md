# GroupDocs.Editor：完整文件編輯 API（適用於 .NET 與 Java）

- **可以批次轉換多個檔案嗎？** 可以，您可以迴圈處理檔案，批次將 PDF / DOCX 轉換為 HTML。  
- **轉換是否安全？** 安全，API 支援受密碼保護的檔案與加密功能。

## 如何使用 GroupDocs.Editor 從 DOCX 提取 HTML

要 **從 DOCX 提取 HTML**，首先使用相應的 `Document` 類別（.NET 與 Java 皆提供）載入 DOCX 檔案，然後呼叫 `ConvertToHtml` 方法。API 會回傳保留樣式、圖片與版面配置的乾淨 HTML。此 HTML 可直接供網頁文件編輯器使用，或依上述 CSS 處理教學進一步處理。

### 標準工作流程
1. **Load the DOCX** – 使用檔案串流或路徑載入。  
2. **Convert to HTML** – 呼叫轉換 API。  
3. **Edit the HTML** – 可選擇使用 CSS 處理或 HTML 操作教學。  
4. **Save back to DOCX** – 編輯完成後，將 HTML 轉回 DOCX，以保留原始格式。

## 如何在 Java 中載入 Excel 檔案（load excel file java）

若需處理試算表，Java API 提供 `SpreadsheetDocument` 載入方法。您可以載入 `.xlsx` 檔案、讀取儲存格值，甚至將活頁簿轉為 HTML 供網頁編輯使用。此功能在建置儀表板或資料驅動的 Web 應用時相當有用。

## 建置網頁文件編輯器（web based document editor）

GroupDocs.Editor 的 HTML 輸出非常適合與任何 JavaScript WYSIWYG 編輯器（例如 TinyMCE、CKEditor）整合。將 DOCX 轉為 HTML 後，使用者即可在瀏覽器中直接編輯內容，編輯完成後再將修改過的 HTML 送回伺服器，重新轉換回原始格式。

## 使用 Java 轉換文件（convert doc to html java）

Java SDK 包含 `DocToHtmlConverter`，可處理 DOC、DOCX 以及其他格式。它會保留內嵌圖片、表格與自訂樣式，確保產生的 HTML 與原始文件外觀完全相同。

## 批次轉換 PDF 與 DOCX 檔案（batch convert pdf docx）

針對大規模遷移，您可以遍歷資料夾中的 PDF 或 DOCX 檔案，於每次迭代呼叫轉換 API。SDK 為執行緒安全，支援平行處理以加速批次作業。

## 在 .NET 中編輯 Word 文件（edit word document .net）

.NET API 提供 `WordProcessingDocument` 類別，讓您修改段落、表格，甚至追蹤變更。編輯完成後，只需一次方法呼叫即可重新匯出為 DOCX 或 PDF。
