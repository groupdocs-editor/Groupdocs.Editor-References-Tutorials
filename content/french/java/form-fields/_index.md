# Créer des formulaires interactifs Java – Édition des champs de formulaire avec GroupDocs.Editor

Dans ce hub, vous découvrirez tout ce dont vous avez besoin pour **créer des solutions basées sur Java pour des formulaires interactifs** avec GroupDocs.Editor. Que vous construisiez une application web centrée sur les documents, un pipeline automatisé de traitement de formulaires, ou que vous ayez simplement besoin de manipuler les champs de formulaire par programme, ces tutoriels vous guident à travers des scénarios réels, étape par étape. Vous apprendrez à modifier, réparer et préserver les données des champs de formulaire tout en assurant une expérience utilisateur fluide et fiable.

## Créer des formulaires interactifs Java – Vue d'ensemble

GroupDocs.Editor for Java offre aux développeurs une API puissante pour charger des documents, travailler avec des champs de formulaire anciens et modernes, et enregistrer les résultats sans perdre l’interactivité. En suivant les guides ci‑dessous, vous pourrez :

* Charger des fichiers Word ou PDF contenant des éléments de formulaire interactifs.  
* Détecter et réparer des collections de champs de formulaire invalides ou corrompues.  
* Définir, lire ou effacer les valeurs des champs par programme.  
* Conserver la mise en page et le style d'origine tout en mettant à jour le contenu du formulaire.
